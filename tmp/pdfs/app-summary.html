Web-Based Discord Clone

Repo summary based on README, frontend source, backend routes, Docker config, and database setup.

What It Is

A browser-based communication app modeled after Discord, with a React frontend, Express backend, and PostgreSQL data store. The repo includes chat, servers/channels, direct messaging, friend workflows, and summary features packaged to run with Docker Compose.

Who It’s For

People who want Discord-style communities and private chats in the browser, including users who need spaces, topic-based channels, DMs, and lightweight catch-up tools.

What It Does

  • Authenticates users with login, registration, JWT-backed sessions, and profile/status updates.
  • Lets users create and manage servers plus text channels.
  • Supports channel messaging, direct messages, replies, edit/delete, and emoji reactions.
  • Handles friend discovery, requests, acceptance/rejection, and DM creation.
  • Supports server invites with pending, accept, and decline flows.
  • Provides message summaries through Manual Summary and “What You Missed” preview features.

How It Works

Frontend: Vite + React app with React Router routes and a shared AppContext that manages auth, selected server/channel/DM, message state, and unread tracking.

Service layer: apiService.ts calls REST endpoints on http://localhost:3001, attaching the stored JWT for protected requests.

Backend: Express mounts auth, servers, channels, messages, direct messages, summaries, users, friends, and invites routes; startup also initializes tables and seeds demo data.

Data: PostgreSQL stores users, servers, memberships, channels, messages, reactions, DMs, friend requests, and server invites. Summary generation uses an in-repo mocked summarization provider; real-time socket transport was not found in repo.
