Outdated Tests and Unimplemented Features

Repo-backed review of where automated coverage lags current behavior and where visible product capabilities are partial, local-only, or not wired through.

What It Is

This is a one-page status snapshot of test coverage drift and incomplete features in the Discord-clone repo. It focuses on evidence visible in current source, test files, reviewer guidance, and backend route wiring.

Who It’s For

The development team seeking to strengthen automated tests, close feature gaps, and align the UI, API surface, and reviewer expectations.

What It Does

  • Test gap Backend integration tests exist, but no frontend automated tests were found in repo.
  • Test gap Reviewer guidance expects reply-to-message behavior, yet messages.test.js does not verify reply creation or reply rendering.
  • Test gap Channel reorder and message search routes exist, but matching automated tests were not found.
  • Feature gap Mobile “Search direct chats...” state is defined in MainLayout.tsx, but no filtering hook-up beyond input state was found.
  • Feature gap Unread/read tracking is marked “local for now” in AppContext.tsx, so read state is not persisted server-side.
  • Feature gap Real-time socket sync was not found in repo; reviewer docs also note navigation/refresh is needed to observe others’ new messages.

How It Works

Frontend: React + React Router uses AppContext for auth, selected server/channel/DM, messages, invites, and local unread state.

UI flow: components such as MessageArea, MessageInput, MessageItem, and ServerSearch call apiService.ts for REST reads/writes and also contain some local fallbacks.

Backend: Express mounts routes for auth, servers, channels, messages, DMs, summaries, users, friends, and invites on top of PostgreSQL tables created at startup.

Data flow: browser state drives REST requests to port 3001, backend validates access, persists rows in PostgreSQL, then returns normalized payloads consumed by the frontend context. Socket transport was not found in repo.

Highest-Value Test Updates

  • Add API tests for replies, message search, channel reorder, and invite-message rendering side effects.
  • Add frontend or E2E coverage for manual summary modal, “What You Missed,” mention picker, and DM search behavior.
  • Reconcile reviewer checklist items with what the current automated suites actually assert.

Not Found In Repo

  • Frontend unit/component test files.
  • Browser E2E test suite.
  • Persistent server-side unread/read receipts.
  • Dedicated real-time WebSocket or Socket.IO layer.
